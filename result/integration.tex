\subsection{Integration: Human-AI Collaboration in an XR Environment}

Building on the XR environment and the human-AI collaboration (HAC) agent developed in prior iterations, this iteration focused on integrating these components into a unified system.

\paragraph{Design}

In a previous XR development iteration, a heads-up display (XrJoystickHud) was implemented to provide real-time feedback on user joystick inputs. In the present iteration, the interface was extended to visualize the AI agent's suggested joystick actions, enabling a direct comparison between user input and agent guidance.

To support this functionality, the structure of the XrJoystickHud was modified to receive data from two emulator components. DroneXrEmulator, this emulator mapped joystick data from the XR environment to the YueInputModule (see \cref{fig:quad_structure}), while the second emulator PidAutoFlightEmulator retrieved joystick signals generated by the PID controller, as illustrated in \cref{fig:hud}.

Furthermore, based on findings from the path-following evaluation iteration, the agent's control output was observed to be unstable. To mitigate this issue, the decision frequency was reduced to one calculation per 100 frames. In addition, a simplified guidance mechanism was introduced in the XrJoystickHud, where only the expected control quadrant was highlighted, as shown in \cref{fig:integration}.

\paragraph{Enactment}

Test play sessions indicated that the agent was generally able to provide correct operational guidance. However, the visual hints were occasionally unstable, reflecting fluctuations in the agent's output. When users relied solely on the HUD guidance, they were not consistently able to perform correct control actions. In addition, some participants reported that the continuous visual feedbacks became distracting over time.

\paragraph{Analysis}

The findings suggest that the agent parameters require further refinement to improve output stability. Moreover, the results indicate a need for adaptive feedback logic. Specifically, when users are already performing correct control actions, the agent should suppress redundant suggestions to reduce potential cognitive overload.
