\subsubsection{Iteration 4: Downgraded White-Box Playground}

\paragraph{Redesign}

In the previous iteration, performance limitations of the target device were identified as a critical constraint. To address this issue, the realistic city environment was removed. The redesigned environment retained only essential elements, including collision boundaries and a predefined flight route, to support goal-directed navigation.

\paragraph{Enactment}

As illustrated in \cref{fig:white-box}, participants operated the quadcopter within a simplified white-box environment. This environment consisted of minimal visual features, all the buildings designed in the prior iteration were removed, leaving only a boxed corridor, focusing on spatial structure and navigational constraints rather than visual realism.

\end{figure}

\paragraph{Analysis}

The simplified environment improved system performance and provided a more stable interaction experience. Playtesting results indicated that participants were able to complete the task using boundary constraints and waypoint guidance. However, some participants showed difficulty maintaining orientation, which led to disorientation and collisions with boundaries.
